Hello everyone
Today I want to tell you what tools I use every day in my development process.


<h3>PHPStorm</h3>

First tool is of course <a href="https://www.jetbrains.com/phpstorm/">PHPStorm</a>. I was using NetBeans around 5 year before I decided to give a try to JetBrains product.
I had a problem with NetBeans now I can't remember what was it. What I like most about PHPstorm is its ability to help in places you never can image it supposed to work. It can interpret `include` and `require` words and open this files
It is easy to navigate through code and project files.


<h3>Git</h3>
<a href="https://git-scm.com/">Git</a> always and everywhere. Now it is difficult to image my life without version control.
It has so much power and I hate myself because I don't use all the powers. And I should devote more time to discover and train all GIT commands.

<h3>SourceTree</h3>
When I started using GIT I felt strong desire to use some nice GUI because built-in GUI wasn't looking good.
And I tried <a href="https://www.sourcetreeapp.com/">SourceTree</a>. This tool covered almost all basic commands GIT provides and it has fancy interface.

<h3>Docker and Vagrant</h3>
I'm using Docker 6 months for now and it saves me a lot of time when testing new tools and techniques
Before Docker I was using <a href="https://www.vagrantup.com/">Vagrant</a> but it usually takes time to create a VM, provision it and create extra packages you need. With <a href="https://www.docker.com/">Docker</a> all you need is internet, 2 clicks and you have Drupal or Joomla up and running on you local VM or VPS

<h3>Digital ocean</h3>
I discovered <a href="https://digitalocean.com">Digital Ocean</a> 6 months ago and it became my default virtual server provider. Setting up and running new VPS take less than minute. You even can run VPS with preinstalled <a href="https://www.docker.com/">Docker</a>.

<h3>Slak + Hubot</h3>
Since me and my friend started working together on <a href="https://skukit.com">skukit.com</a> we needed tool to communicate with each other and the tool which can gather data from all our tools like scrutinizer, <a href="https://travis-ci.com/">travis-ci</a>, <a href="https://asana.com/">asana</a>. Slack fits perfect in this description. We even run <a href="https://hubot.github.com/">Hubot</a> for extra task but for now don't use often.

<h3>Travis-ci</h3>
I think no need to tell you what is it. If you are passionate about testing as I am you are already know it

<h3>Github</h3>
Best tool to store open source projects

<h3>Postman</h3>
Because oа our project planned in API-first design in mind the best tools we can find was <a href="http://www.getpostman.com/">Postman</a>.
Usually we use swagger-ui but when API is under maintenance Postman comes in handy.

<h3>Swagger</h3>
I chose swagger because it has handy tools like <a href="http://swagger.io/swagger-ui/">swagger-ui</a>, <a href="http://swagger.io/swagger-editor/">swagger-editor</a>, and <a href="http://swagger.io/swagger-codegen/">swagger-codegen</a>. It saves a lot of time.

<h3>Atom</h3>
Use it as a text editor. When I was on Windows I used to be using notepad++ but for Mac I decided to stick with Atom.  Cannot tell much about it, but it covers all my need for text editor.

<h3>Asana</h3>
As many of you I worked with tens of project management tools and bug trackers like: <a href="http://www.jetbrains.com/youtrack">Youtrack</a>, <a href="http://www.thebuggenie.com">The Bug Genie</a>, <a href="http://www.redmine.org/">Redmine</a> all of them has its own pros and cons. But currently I manage my project task in <a href="https://asana.com/">Asana</a> and it turned to be a very good tools with mobile app. What it needs more is better integration with external tools.
